# Gentoo

Gentoo is a model-agnostic method for fusing geodetic data streams, one that does not require any reference to a particular geophysical model. Data fusion, often referred to as sensor fusion, is common in meteorology, medical imaging, and aviation, and seeks to integrate two or more data streams into a single derived data product, isolating the signal shared by all inputs while exploiting their complementary strengths (e.g., Castanedo, et. al. 2013). The data fusion method we propose is based on Gaussian process regression and a semi-parametric latent factor model. Our use of Gaussian process regressions follows from (Hines and Hetland, 2017), who used a Gaussian process regression framework to interpolate GNSS data in space and time. While their method, coined PyGeoNS (*Py*thon-based *geo*detic *n*etwork *s*moother), derived a spatio-temporal, smooth deformation field, yielding time-varying strain-rates, it was not a data fusion. Henceforth, we  refer to our method as "Gentoo", where "gen"  derives from "*ge*odetic *n*etwork smoother" following Hines and Hetland (2017), and "too" loosely referencing the second evolution of PyGeoNS. At present, Gentoo only considers time-invariant deformation, either static offsets or secular velocities. Gentoo does not require the various data streams to be collocated, and results in a contiguous estimate of the three component surface deformation field, complete with corresponding estimates of the full uncertainties for each component.

This version of Gentoo reflects the code base used to generate the results found in the original publication "Geodetic Data Fusion using Multi-Output Sparse Gaussian Processes."
